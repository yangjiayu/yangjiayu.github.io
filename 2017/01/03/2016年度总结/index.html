2016年度总结

2017-01-03

2016

2016年算是成长较快的一年，应该是说毕业五年之内成长都是很快的，但是感觉还是时间不够用，毕竟同龄人之间牛逼的太多了。

OC

对于oc来说，进步空间还是蛮大的，年初的时候同事跳槽离职，整个公司的项目都是我抗着的，毕竟App的用户量在那，如果做不好不仅自己觉得不舒服，用户感觉坑了也会骂你。所以说这个时间点是OC成长最快的时候，项目是基于ReactiveCocoa框架的，原本说是要在招聘一个开发，但是接连发了两个版本之后，公司面临转型了，后面的技术栈转移到了ReactNative。再到后来年末的时候，发觉reactnative的各种坑，感叹不该踏入这个旋涡啊，这个期间看了大量的oc的开源库，也算是经过半年的时间之后把oc又补了一下。

React系列

公司转型的时候，技术栈转移到了reactnative，这个时候rn刚刚发布了android版本没有多久，好像是0.25版本，到现在已经0.39了。刚开始接触的时候发现是真的不错。一套代码可以同时在android和ios端同时跑起来。后来发现了各种各样的性能问题，比如listview的row（cell）不复用，android端调用原生代码会莫名奇妙的crash，UI性能没有调优了。到现在reactnative还没有发布1.0版本，变化还是很大的，所以还是蛮期待reactnative的正式版本。
中间做了一份基于react的web项目，感觉上手很容易。真的是Learn once, write anywhere。
reactnative是基于react的封装了一套映射原生的UI控件，reactnative的module控件都可以在native找到封装的api。参考了bang的blog。

JavaScript

原来只是比较浅显的写过一点JavaScript，趁着reactnative又把js重拾了一遍。这个期间学到了大量的框架模式，函数式编程、面向组合子编程，对代码的编程范式有一个很大的转变。JavaScript是一门很优秀的语言，比oc和swift写起来舒服自由的多。

读书和生活

今年大部分看的都是技术类的书籍，也参加了一些技术的分享会。其他的方面就看了一下《时间简史》和《人类简史》，陪着女朋友参加了一个noms全国运营大会，墨迹天气的产品经理李总监分享的特别好，遗憾的是分享完没有找到人，没有加上联系方式，继续效仿学习的机会就这样丢失了。现在好好努力，以后也可以分享者的姿态去普及小弟了。

工作

2015年末加入了一家创业公司，看起来发展的不错，没想到还是继续呆了一年的时间。遇到了国家政策突变，业务转型，到现在过去了一年的时间才刚刚业务做起来，这一年兢兢业业，但是没想到做起来KPI，本来就不高的工资还扣掉不少，哎。现在技术和业务管理继续都应该没有什么成长了，明年考虑换一家公司。

2017年展望

  • 技术层面要继续去挖掘，不仅是限于oc、js，尝试掌握一门后端开发语言，例如python。
  • 好好整理一下blog。2016年有点懒惰，没有整理过blog，希望在2017年做好，做到每个月都有高质量的产出。毕竟笔记和blog不一样，blog可以是笔记，但是必须做精，应为不仅自己看，其他人也会看。
  • 继续锻炼的还有很多，提高一下自己的情商，提高一下英语的听力。
  • 比你厉害的人不可怕，可怕的是厉害的人比你还努力。2017年，新的一年继续加油💪。
