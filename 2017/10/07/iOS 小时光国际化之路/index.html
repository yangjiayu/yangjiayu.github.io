iOS国际化之路

2017-10-07 • 分类: ios

前提

国际化分步：

  • 添加国际化需要的配置语言
  • Info.plist文件中文本国际化
  • 代码中文本国际化。
  • 代码中图片国际化。
  • storyboard/xib国际化
  • App内切换语言环境

步骤

添加国际化需要的配置语言

  1. 点击XCode，Project -> Info -> Localizations 添加需要国际化的语言

Info.plist文件中文本国际化

  1. 新建Strings File 文件，文件必须取名为InfoPlist.strings文件。
  2. 选中InfoPlist.strings文件，在XCode的File inspection中点击Localize，选择需要国际化的语言。
  3. 在InfoPlist.strings文件的下拉列表里，分别进行Info.plist文件中文本的国际化，例如CFBundleDisplayName,NSCameraUsageDescription.

代码中文本国际化。

  1. 新建String File文件，文件必须取名为Localizable.strings文件。
  2. 选中Localizable.strings文件，在XCode的File inspection中点击Localize，选择需要国际化的语言。
  3. 在Localizable.strings文件的下拉列表里，分别进行代码中字符串文本的国际化。
  4. XCode提供的全局搜索功能，Command + shift + F中，选中Replace -> Regular Expression, 查找代码中文本进行替换，replace查找支持正则表达。在查找中，查找正则@"[^"][\u4E00-\u9FA5]+[^"\n]?", 使用宏替换NSLocalizedString($1, nil)($1代表着前面的正则).

国际化创建strings文件时，会在workspace的同级目录下创建相应的文件夹，例如：en.lproj, zh-hant.lproj。然后在相应的目录下会有国际化的strings文件，对strings中的文件进行国际化。

代码中图片国际化

图片的国际化有两种方式。

  • 以文件形式存在的图片，可点击XCode的File inspection，点选Localize不同的国际化方式，替换不同的国际化图片。
  • 在xcassets资源文件中的图片，类似于文本的国际化方式，将图片的字符串用NSLocalizedString包裹，在相应的文件中进行替换。可新建相应的图片字符串文件

storyboard/xib国际化

storyboard/xib的国际化也有两种方式

  • 点选XCode的File inspection -> Localize, 选择不同的语言类型，然后在SB/xib的可选下拉框中，进行国际化语言的替换。(存在一个问题，1、不利于统一国际化的管理。 2、如果做App内切换语言，这块没有办法做，只能是跟着系统的默认语言走。)

  • 将SB/Xib中的所有文字和图片，在相应的.m文件中做属性，利于属性进行NSLocalizedString的宏包裹。
