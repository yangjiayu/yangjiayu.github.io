创建.a静态文件问题

2018-03-05

前提

今天牵扯到一个API加密算法的问题，要把加密算法和密钥放到静态文件里面，防止代码泄漏造成的api加密破解，攻击。

创建.a 静态文件

新建工程 — 选择Cocoa Touch Static Library — 相应的代码 — Targets, Build Phases, Copy Files 选择想要暴露出来的文件 — Targets和Project, Build Active Architecture Only 设为NO。— 分别在模拟器下和真机下run。 — 左侧目录下的products就回有libXXXX.a 文件由红(未生成)变成黑(已生成) — Show In Finder

Build Active Architecture Only这个字段表明：只在当前的处理器下进行编译，当进行打包时需要兼容所有的处理器。

iPhone5s及以上：arm64(真机) 和 x86_64(模拟器)
iPhone5及一下：arm7(真机) arm7v(真机)和i386(模拟器)

合成

上一步会得到两种.a文件，一种是支持模拟器的，一种是支持真机的。
调用命令: lipo -create ‘文件路径/libXXXX.a’ ‘文件路径/libXXXX.a’ -output ‘文件路径/libXX.a’ 生成一个同时支持模拟器和真机的.a文件。

遇到问题

整个开发环境在xcode 9+, Build Active Architecture Only及时设为NO，生成的libXXXX.a文件通过命令 lipo -info libXXXX.a 查看也只支持一种 arm64或者x86_64的格式。

具体的原因没有细究，猜想应该是默认的支持的系统版本太高了，新建工程默认是支持ios11+的，目前支持ios11+的全是支持arm64架构的。

解决： 将支持的版本调低一点就好了
Targets — Build Settings — iOS Development Target设为ios8.0
